7月になりました。5月12日に始めたこの日記（ブログ）では、5月に9本、6月に6本の記事を書きました。

## 盆栽のように

私は本業でもウェブサイトをつくるお仕事をしています。

仕事では、様々な技術を用途に応じて採用しますが、趣味プロダクトでは、後で述べるような好きなものをいくらでも導入することができます。

誰に頼まれてやっているわけでもない、締め切りもないものです。好きな時に、インクリメンタルに、デザインや機能を変えていったりできます。

このブログも、最初はシンプルな文字だけのデザインでしたが、数週間経って、サムネイル画像を表示する形に変えました。この記事をあなたが読んでいる頃には、また変わっているかもしれません:

## 好きな技術を好きなだけ: Astro, UnoCSS, Vercel

この日記のコードおよびコンテンツ、編集履歴は、全て以下のページ（GitHubレポジトリ）で確認できます:

[sorami/costep: 科学技術コミュ日記](https://github.com/sorami/costep)

実装には「Astro」というWebフレームワークを使っています。2022年にv1.0がリリースされました。Next.jsが複雑なWebアプリの構築へ適しているのに対し、Astroはコンテンツをベースとしたウェブサイト（ブログやランディングページなど）へフォーカスしています。私としては「まさにこういうのが欲しかったんだよ！」という気持ちで、使いやすく、昨年末くらいから積極的に利用しています:

[Astro](https://astro.build/)

ちなみに個人的には、以前は静的サイトジェネレーターの[Eleventy（11ty）](https://www.11ty.dev/)を使っていましたが、そちらはNunjucksやHAML、といったテンプレート言語を使うものでした。一方でAstroは、よりモダンなWeb開発のフレーバーです。また、ReactやVue.js、Svelteといったフレームワークのコンポーネントをそのまま利用することもできます:

<YouTube videoId="gxBkghlglTg" />

また、↑のような埋め込みを容易にするため、Astroでコンポーネントを作成し、それをMDXで利用しています:

[Markdown for the component era | MDX](https://mdxjs.com/)

[Markdoc](https://markdoc.dev/)にも興味があるのですが、そのタイミング（2023年5月ごろ）はまだAstroの[Content Collectionsという機能](https://docs.astro.build/en/guides/content-collections/)にMarkdocが正式対応していなかったようなので、MDXを使っています:

- [Markdoc support by bholmesdev · Pull Request #508 · withastro/roadmap](https://github.com/withastro/roadmap/pull/508)
- [@astrojs/markdoc 🚀 Astro Documentation](https://docs.astro.build/en/guides/integrations-guide/markdoc/)

スタイルには、UnoCSSというアトミックCSS（ユーティリティファーストCSS）フレームワークを使っています（[Tailwind CSS](https://tailwindcss.com/)のようなものです）。個人的にかなり好きで、手に馴染みます:

- [UnoCSS: The instant on-demand Atomic CSS engine](https://unocss.dev/)
- [Reimagine Atomic CSS](https://antfu.me/posts/reimagine-atomic-css)（antfu, 2021）

デプロイにはVercelを使っています。無料プランで十分使えています。シンプルで、開発者体験が良くて好きです:

[Vercel: Develop. Preview. Ship. For the best frontend teams](https://vercel.com/)

ぼちぼちやっていきます 🧑‍🌾
